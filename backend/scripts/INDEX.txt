================================================================================
ANALYSIS COMPONENTS (5 Queries)
================================================================================

QUERY 1: PROCESS QUALITY RANKING
├─ Purpose: Which processes have best teams/management?
├─ Output: Process name, quality rank, variance, tier, excellence rate
├─ Key Metric: Quality rank (1=best) and variance (<8=stable)
├─ Use For: Resource allocation, identify training models
└─ Example: "Process_A ranks #1 with 86% quality and 7.8 variance (ELITE)"

QUERY 2: FATIGUE CYCLE ANALYSIS
├─ Purpose: When does quality decline due to agent fatigue?
├─ Output: Day-of-week, hour, call volume, quality, fatigue factor
├─ Key Metric: Fatigue factor (1=best mid-week, 3=worst weekend)
├─ Use For: Shift scheduling, call routing, break optimization
└─ Example: "Tuesday 9-11 AM shows 84% quality (optimal); Friday 6 PM shows 72%"

QUERY 3: SHIFT TIMING OPTIMIZATION
├─ Purpose: Which shift window has best performance?
├─ Output: Shift, quality, call volume, agents, recommended workload
├─ Key Metric: Peak shift classification (primary, secondary, avoid)
├─ Use For: Route complex work to optimal windows
└─ Example: "Morning peak: 84% quality (complex work); Evening: 76% (routine)"

QUERY 4: PROCESS + SHIFT MATRIX
├─ Purpose: Which process performs best in which shift?
├─ Output: Process, shift, quality, team size, allocation priority
├─ Key Metric: Allocation priority (primary, secondary, avoid)
├─ Use For: Roster planning, agent allocation strategy
└─ Example: "Process_A + Morning: 87% (primary); Process_A + Night: 68% (avoid)"

QUERY 5: TEAM COMPOSITION
├─ Purpose: Right-size teams per process/shift?
├─ Output: Process, shift, team size, quality, load status, recommendation
├─ Key Metric: Team size vs quality correlation
├─ Use For: Org restructuring, span of control optimization
└─ Example: "Teams 6-10 agents: 82% quality (optimal); 16+ agents: <70% (crisis)"

KEY METRICS & THRESHOLDS
================================================================================

QUALITY SCORE (%)
  85%+   = ELITE          → Expand allocation
  80-85% = HIGH           → Maintain & monitor
  75-80% = MEDIUM         → Monitor & coach
  70-75% = DEVELOPING     → Intervention program
  <70%   = AT_RISK        → URGENT restructure

QUALITY RANK
  Rank 1-3   = Top tier   → Best practices model
  Rank 4-5   = High tier  → Maintain current state
  Rank 6-10  = Mid tier   → Opportunity for improvement
  Rank 11+   = At-risk    → Action required

VARIANCE (Consistency)
  <8      = STABLE        → Predictable, low variation
  8-12    = MODERATE      → Typical, acceptable
  >12     = VOLATILE      → Investigate consistency issues

FATIGUE FACTOR
  1       = OPTIMAL       → Best time for complex work
  2       = MODERATE      → Mixed work acceptable
  3       = HIGH FATIGUE  → Routine/automation only

TEAM SIZE
  6-10    = OPTIMAL       → Best quality 82-85%
  11-15   = STRETCHED     → Quality declining 77-80%
  16-20   = OVERLOADED    → Quality <75%
  21+     = CRISIS        → Immediate restructure needed

Q3: When is the best time for complex work?
A: Tuesday-Thursday, 9-11 AM or 2-4 PM
   Quality: 83-84% (optimal)
   Avoid: Lunch hour (12-1 PM), Friday, weekends

Q4: How do I know if my team is too large?
A: Red flags: >14 agents, quality <78%, variance >12%

A: Options:
   1. Route easier work on Friday
   2. Shorten Friday shifts (6 hrs vs 8)
   3. Offer Friday incentive
   4. Rotate Monday-off for Friday teams
